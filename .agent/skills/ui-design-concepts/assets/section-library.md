# Section Library - Pre-designed Concepts

Ready-to-use section ideas for building complete pages. Mix and match sections to create full experiences.

---

## How to Use This Library

Each section has:
- **Purpose**: What it does
- **Visual Concept**: How it looks
- **Layout Idea**: Composition approach
- **Content Strategy**: What goes inside
- **Variants**: Different takes on the concept

**Combine sections to build pages:**
```
Landing Page = Hero A + Features C + Social Proof B + Pricing A + CTA B + Footer
Product Page = Breadcrumbs + Hero Split + Tabs + Related + Trust Footer
Dashboard = Stats Bar + Main Chart + Activity Feed + Quick Actions
```

## HERO SECTIONS (10 Variants)

### Hero A: Centered with Floating Card
**Purpose**: Focus attention, single clear CTA
**Visual**: Large headline centered, supporting text below, prominent CTA button, optional floating card/mockup image overlay
**Layout**: Centered vertically and horizontally, background can be gradient, image, or video
**Content**: Headline (benefit-focused), subtext (2-3 lines), primary CTA, optional secondary CTA
**Vibe**: Clean, confident, modern SaaS
**Best For**: SaaS products, services, B2B

### Hero B: Split Screen (50/50)
**Purpose**: Show product alongside message
**Visual**: Text content on left, large visual (image/video/mockup) on right
**Layout**: 50/50 split on desktop, stacked on mobile (image first)
**Content**: Headline, longer description (3-4 lines), bullets or benefits, CTA
**Vibe**: Balanced, professional
**Best For**: Products with strong visuals, apps, physical products

### Hero C: Full-screen Video Background
**Purpose**: Immersive brand experience
**Visual**: Full-screen background video (slow motion, branded), dark overlay for text contrast, centered content
**Layout**: Absolutely centered, video covers full viewport
**Content**: Short punchy headline, minimal text, single strong CTA, scroll indicator at bottom
**Vibe**: Premium, immersive, bold
**Best For**: Fashion, luxury, creative agencies

### Hero D: Asymmetric with Floating Elements
**Purpose**: Stand out, creative expression
**Visual**: Off-center text, floating product images or shapes, diagonal elements, lots of white space
**Layout**: Asymmetric grid, elements positioned deliberately "wrong"
**Content**: Headline (can be oversized), minimal supporting text, CTA, scattered trust badges or small visuals
**Vibe**: Creative, modern, unconventional
**Best For**: Creative agencies, portfolios, innovative startups

### Hero E: Minimal Text-Only
**Purpose**: Let the message speak
**Visual**: Just text, no images. Extra-large typography, lots of breathing room
**Layout**: Centered, maybe slight left-align, generous margins
**Content**: Headline (extra bold), subtext, CTA, nothing else
**Vibe**: Confident, editorial, premium
**Best For**: Consultancies, high-end services, thought leadership

### Hero F: Interactive Demo Preview
**Purpose**: Show product immediately
**Visual**: Headline above, live/interactive product demo below (iframe, embedded app, or realistic mockup with hover effects)
**Layout**: Text section normal height, demo section takes remaining viewport
**Content**: Headline, one-liner explanation, small CTA, demo/preview dominates
**Vibe**: Product-first, transparent, engaging
**Best For**: SaaS tools, apps, interactive products

### Hero G: Multi-layer Parallax
**Purpose**: Create depth, guide attention
**Visual**: Multiple layers (background image, middle elements, foreground text) that move at different speeds on scroll
**Layout**: Full-screen with layered elements, content centered
**Content**: Headline, subtext, CTA, decorative shapes or images in layers
**Vibe**: Dynamic, modern, storytelling
**Best For**: Marketing sites, creative projects, portfolios

### Hero H: Stats-Driven
**Purpose**: Prove credibility immediately
**Visual**: Headline and CTA prominent, but large stats displayed around them (users, reviews, savings, etc.)
**Layout**: Centered main content, stats positioned as accent elements around it
**Content**: Headline, CTA, 3-4 big impressive numbers with labels
**Vibe**: Data-backed, trustworthy, results-focused
**Best For**: B2B, tools with proven track record, marketplaces

### Hero I: Carousel / Rotating
**Purpose**: Show multiple messages or products
**Visual**: Full-screen slides that rotate (auto or manual), each with own headline/image/CTA
**Layout**: Full viewport slides, dots or arrows for navigation
**Content**: 3-5 slides, each with unique message/visual/CTA
**Vibe**: Dynamic, content-rich
**Best For**: E-commerce, news, multi-product companies
**Note**: Use sparingly, can hurt conversion if overused

### Hero J: Story-Driven Scroll
**Purpose**: Guide user through narrative
**Visual**: Full-screen sections that appear on scroll, each advancing the story, sticky elements that animate
**Layout**: Vertical scroll-based sections, scroll hijacking or snap scroll
**Content**: Sequential story beats, visuals that support narrative, CTA at end
**Vibe**: Immersive, cinematic, storytelling
**Best For**: Product launches, brand stories, case studies

## FEATURE SECTIONS (10 Variants)

### Features A: Grid with Icons
**Purpose**: Show multiple features at a glance
**Visual**: 3-4 column grid, icon at top of each card, title, short description
**Layout**: Responsive grid (4 col → 2 col → 1 col)
**Content**: Icon (simple, line-style), feature name, 2-3 sentence description
**Vibe**: Clean, organized, informative
**Best For**: SaaS, products with many features

### Features B: Bento Grid
**Purpose**: Visual hierarchy, highlight key feature
**Visual**: Mixed-size grid where important features get more space, can include images/graphics
**Layout**: CSS Grid with varied cell sizes, asymmetric but balanced
**Content**: 5-8 features, main one is 2x size with richer content, others are standard cards
**Vibe**: Modern, dynamic, Apple-esque
**Best For**: Products wanting to highlight one killer feature

### Features C: Alternating Rows
**Purpose**: Detailed explanation of each feature
**Visual**: Feature image left, text right. Next row: text left, image right. Alternating pattern.
**Layout**: Two-column alternating, full-width sections
**Content**: Feature image/mockup, headline, longer description (paragraph), optional list of sub-features
**Vibe**: Thorough, professional, easy to scan
**Best For**: Complex products, B2B, detailed features

### Features D: Tabs
**Purpose**: Show many features without overwhelming
**Visual**: Horizontal tabs at top, clicking changes the main content area below (image + description)
**Layout**: Tab navigation bar, large content area below
**Content**: 5-10 feature categories in tabs, each with own visual and description
**Vibe**: Interactive, organized, tech-forward
**Best For**: Products with feature categories, platforms with multiple tools

### Features E: Accordion List
**Purpose**: Compact presentation, user explores
**Visual**: Vertical list of feature names, clicking expands to show details, one open at a time
**Layout**: Single column, stacked items
**Content**: Feature title always visible, expandable content with description and optional image
**Vibe**: Minimal, exploratory, mobile-friendly
**Best For**: Long feature lists, FAQ-style, mobile-first products

### Features F: Icon + Title Only (Minimal)
**Purpose**: Quick overview, links to detail pages
**Visual**: Simple icon, feature name below, arranged in grid, very minimal
**Layout**: Grid (4-6 columns), compact
**Content**: Just icon and title, clicking goes to detail page
**Vibe**: Super clean, confident, assumes interest
**Best For**: Well-known products, feature directory pages

### Features G: Comparison Table
**Purpose**: Compare tiers or options
**Visual**: Table format, your options across columns, features down rows, checkmarks/X for availability
**Layout**: Responsive table, horizontal scroll on mobile or card-based
**Content**: Feature list, 2-4 options to compare, visual indicators (✓ / ✗ / partial)
**Vibe**: Analytical, transparent, decision-focused
**Best For**: Pricing tiers, product variants, competitive comparison

### Features H: Timeline/Process
**Purpose**: Show steps or progression
**Visual**: Vertical or horizontal timeline, each point has icon, title, description
**Layout**: Linear progression with connecting line
**Content**: 3-5 steps, each with visual marker, title, description
**Vibe**: Story-driven, process-oriented, educational
**Best For**: How it works, onboarding process, implementation steps

### Features I: Video Showcase
**Purpose**: Demo features in action
**Visual**: Grid or list of feature videos (looping demos), title overlay, click to expand
**Layout**: Video thumbnails in grid, modal or expanded view on click
**Content**: Short looping videos (5-15s each) showing feature in use, title overlay
**Vibe**: Interactive, proof-driven, modern
**Best For**: SaaS with visual features, tools, apps

### Features J: Interactive Module
**Purpose**: Let user experience feature
**Visual**: Embedded mini-app or interactive element showing feature, explanation alongside
**Layout**: Feature module takes center, explanation text around it
**Content**: Working example or simulation, interactive controls, explanatory text
**Vibe**: Hands-on, impressive, engaging
**Best For**: Developer tools, visual products, unique interactions

## SOCIAL PROOF (8 Variants)

### Social Proof A: Logo Cloud
**Purpose**: Show brand credibility
**Visual**: Grid of company logos (grayscale), headline above like "Trusted by..."
**Layout**: Centered, logos in rows, responsive
**Content**: 8-16 recognizable brand logos, brief headline
**Vibe**: Credible, established, enterprise-ready

### Social Proof B: Testimonial Cards
**Purpose**: Personal recommendations
**Visual**: Cards with quote, person photo, name, title/company
**Layout**: 3-column grid (or slider on mobile)
**Content**: Quote (2-4 sentences), headshot, name, role, optional company logo
**Vibe**: Personal, relatable, trust-building

### Social Proof C: Stats Bar
**Purpose**: Quick impressive numbers
**Visual**: Horizontal bar with 3-4 big numbers and labels
**Layout**: Single row, evenly spaced, can be sticky
**Content**: Big numbers (users, revenue, satisfaction, etc.) with short label below each
**Vibe**: Data-driven, impressive, factual

### Social Proof D: Featured Testimonial (Large)
**Purpose**: Highlight one powerful story
**Visual**: Full-width section, large quote, prominent person, background image optional
**Layout**: Centered or split-screen, generous spacing
**Content**: Longer quote (3-5 sentences), large photo, name/title, optional company context
**Vibe**: Impactful, story-driven, premium

### Social Proof E: Video Testimonials
**Purpose**: Show real people
**Visual**: Video thumbnails in grid, play inline or modal
**Layout**: Grid of video cards with play button overlay
**Content**: 15-60s customer videos, thumbnail with play button, name/company underneath
**Vibe**: Authentic, engaging, high-trust

### Social Proof F: Case Study Cards
**Purpose**: Detailed success stories
**Visual**: Cards with company logo, results numbers, brief description, "Read case study" link
**Layout**: 2-3 column grid
**Content**: Company logo, headline result, 2-3 key metrics, brief description, CTA to full case study
**Vibe**: B2B, results-focused, professional

### Social Proof G: Review Aggregate
**Purpose**: Show overall satisfaction
**Visual**: Large star rating, number of reviews, excerpt from top reviews
**Layout**: Centered rating display, reviews below in list or grid
**Content**: Average star rating (large), total reviews count, 3-5 recent reviews with stars/excerpt
**Vibe**: Consumer-friendly, transparent, data-backed

### Social Proof H: Media Mentions
**Purpose**: Show press coverage
**Visual**: "As featured in" headline, logos of publications (New York Times, TechCrunch, etc.)
**Layout**: Single row of logos, sometimes with quotes from articles
**Content**: Publication logos, optional headline quotes
**Vibe**: PR-focused, newsworthy, established

## PRICING (6 Variants)

### Pricing A: Card Tiers
**Purpose**: Clear tier comparison
**Visual**: 3 pricing cards side-by-side, middle one highlighted/elevated
**Layout**: 3 columns (stacked on mobile), center card larger/emphasized
**Content**: Plan name, price, billing period, feature list, CTA button per card
**Vibe**: Standard, clear, conversion-optimized

### Pricing B: Table View
**Purpose**: Detailed feature comparison
**Visual**: Plans as columns, features as rows, checkmarks show inclusion
**Layout**: Responsive table, horizontal scroll or stacked on mobile
**Content**: All features listed, visual indicators (✓ / ✗) for each plan
**Vibe**: Analytical, transparent, for researchers

### Pricing C: Slider/Toggle
**Purpose**: Custom pricing based on usage
**Visual**: Slider to adjust usage (users, volume, etc.), price updates dynamically
**Layout**: Centered calculator interface, plan cards below
**Content**: Interactive slider, calculated price display, feature breakdown
**Vibe**: Flexible, usage-based, modern SaaS

### Pricing D: Simple Single Price
**Purpose**: One option, no confusion
**Visual**: Large price display, feature list below, single CTA
**Layout**: Centered, vertical stack
**Content**: Price (prominent), billing info, comprehensive feature list, one CTA
**Vibe**: Confident, simple, no games

### Pricing E: FAQ-Style
**Purpose**: Address concerns while showing price
**Visual**: Pricing info at top, FAQ accordion below
**Layout**: Pricing summary, then expandable Q&As
**Content**: Basic pricing info, 5-10 common questions answered
**Vibe**: Educational, transparent, concern-addressing

### Pricing F: Feature-Based Cards
**Purpose**: Choose by need, not tier name
**Visual**: Cards organized by use case ("For Freelancers", "For Teams", etc.)
**Layout**: 2-4 columns, descriptive card headers
**Content**: Use case headline, price, features relevant to that persona, CTA
**Vibe**: Persona-focused, relatable, helpful

## CTA SECTIONS (6 Variants)

### CTA A: Centered Minimal
**Purpose**: Final push to action
**Visual**: Centered headline, subtext, single button, lots of white space
**Layout**: Vertically centered, maximum simplicity
**Content**: Action-focused headline, one-line benefit, CTA button
**Vibe**: Clean, confident, pressure-free

### CTA B: Split with Image
**Purpose**: Visual reinforcement
**Visual**: Text + CTA on left, image/illustration on right
**Layout**: 50/50 or 60/40 split
**Content**: Headline, benefits, CTA, supporting visual
**Vibe**: Balanced, persuasive, visual

### CTA C: Gradient Background
**Purpose**: Stand out, create energy
**Visual**: Vibrant gradient background, white text, prominent button
**Layout**: Full-width section with color, centered content
**Content**: Headline, subtext, CTA, optional small print
**Vibe**: Energetic, modern, attention-grabbing

### CTA D: With Input (Email Capture)
**Purpose**: Collect contact before commitment
**Visual**: Headline, email input field, submit button (inline)
**Layout**: Centered, horizontal input + button combo
**Content**: Headline, email field, submit button, privacy note
**Vibe**: Lead-gen focused, newsletter-style

### CTA E: Multi-Option
**Purpose**: Offer choices
**Visual**: Headline, then 2-3 different action buttons
**Layout**: Centered headline, horizontal button row below
**Content**: Headline, multiple CTA options (buy, demo, contact, etc.)
**Vibe**: Flexible, exploratory, customer-choice

### CTA F: Urgency-Based
**Purpose**: Create FOMO
**Visual**: Headline with urgency element (countdown, limited spots, etc.), prominent CTA
**Layout**: Centered, countdown or urgency indicator prominent
**Content**: Time-sensitive headline, countdown/scarcity indicator, urgent CTA
**Vibe**: High-pressure, conversion-focused
**Note**: Use ethically, real scarcity only

## Usage Guide

**Building a landing page:**
```
1. Choose hero type (consider your message)
2. Add features section (show value)
3. Include social proof (build trust)
4. Add pricing (if applicable)
5. End with CTA
6. Comprehensive footer
```

**Building product page:**
```
1. Breadcrumbs
2. Split product display (E-commerce D)
3. Product details tabs
4. Related products carousel
5. Reviews section
6. Trust footer
```

**Building dashboard:**
```
1. Sidebar navigation (Dashboard F)
2. KPI cards at top (Dashboard A)
3. Main chart or content
4. Activity feed or quick actions
5. Data table if needed
```

**Mix and match freely!** These are concepts, not rigid rules. Combine, modify, and create your own variations.
